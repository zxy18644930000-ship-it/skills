# 建筑劳务功效管理系统

**背景**: 用户同时经营建筑劳务公司，主做承包劳务（如北京外墙旧改项目，千人规模）。

**核心痛点**:
- 很多活包不出去（没人愿意按量承包），只能按日薪雇工人
- 按日薪就无法量化功效，大量工人打卡后消失骗工时
- 工人甚至集体告状要求涨工资，缺乏数据反驳
- 现有商业方案太贵，且不接地气

**产品定位**: 自用 + 商业化SaaS，面向建筑劳务行业

**技术方案演变**:
- ~~原方案: GPS硬件定位器（工牌式），1000人×100元=10万硬件成本~~
- **新方案 (2026-03-15确定)**: 直接用工人手机APP采集GPS，零硬件成本

**MVP APP已完成 (2026-03-15)**:
- 技术栈: Flutter 3.41 + Dart
- 项目路径: `~/Projects/worker_tracker/`
- APK: `~/Desktop/工地考勤.apk` (45MB)
- 功能: 工人编号输入 → 开始/停止打卡 → 每2分钟GPS采集 → 本地SQLite存储 → 模拟定位检测
- 目标设备: 华为Mate60 Pro+ (HarmonyOS 4.2.0, 兼容Android APK)

**开发环境**:
- Flutter SDK: `~/flutter/` (3.41.4 stable)
- Android SDK: `~/Library/Android/sdk/` (SDK 36)
- Java: `/opt/homebrew/opt/openjdk@17`
- 环境变量已写入 `~/.zshrc`

**功效模型核心**：`功效 = 区域进度增量 / 该区域当天投入工时`
- 同区域不同班组横向对比 → 人均功效差异
- 长期积累个人跨班组表现 → 个人功效评分

**下一步**:
- 管理端后台 (Python + Web): 接收工人GPS数据、围栏设置、工时统计
- 地图可视化: 高德地图API展示工人位置
- 进度拍照: 班组长任务打点+拍照
- ML模型: 积累3-6个月数据后训练行为模式识别
