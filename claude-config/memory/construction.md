# 建筑劳务功效管理系统

**背景**: 用户同时经营建筑劳务公司，主做承包劳务（如北京外墙旧改项目，千人规模）。

**核心痛点**:
- 很多活包不出去（没人愿意按量承包），只能按日薪雇工人
- 按日薪就无法量化功效，大量工人打卡后消失骗工时
- 工人甚至集体告状要求涨工资，缺乏数据反驳
- 现有商业方案太贵，且不接地气

**产品定位**: 自用 + 商业化SaaS，面向建筑劳务行业

**技术方案（已讨论确定）**:
1. **数据采集层**：GPS定位器（工牌式）+ 班组长任务打点拍照 + 工种标签
2. **功效模型核心**：`功效 = 区域进度增量 / 该区域当天投入工时`
   - 同区域不同班组横向对比 → 人均功效差异
   - 长期积累个人跨班组表现 → 个人功效评分
3. **围栏+工时**：GPS围栏自动计算有效在场时长，离场超15分钟标记
4. **两阶段实施**：
   - 第一阶段（MVP）：GPS围栏 + 工时统计 + 进度拍照 + 规则引擎（1-2个月）
   - 第二阶段：积累3-6个月GPS+产出配对数据后，训练ML模型识别行为模式
5. **技术栈**：Python后端 + Web管理后台 + 工人端小程序

**成本估算**: 1000人×100元定位器=10万硬件 + 1万/月平台费

**项目状态**: 规划讨论阶段，待选定GPS硬件后启动开发
