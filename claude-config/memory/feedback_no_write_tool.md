## 事故

2026-03-08 写入的智能腿序代码（executor.py）在后续对话中被 Write 工具整文件覆盖丢失。覆盖后又没有 git commit，导致代码悄悄丢了，用户3月12日才发现。

## 规则（必须遵循）

1. **修改现有文件永远用 Edit 工具**（局部替换），绝对禁止用 Write 覆盖已有代码文件
2. **每次改完代码立刻 `git commit`**，不留未提交修改在工作目录
3. **改文件前先 Read 相关段落**，确认不会遗漏已有逻辑
4. Write 工具只用于创建全新文件

## 事故影响

- 智能腿序代码丢失，用户花大量时间重复调试
- 用户非常愤怒，认为同一件事反复做了好几次
- 这是实盘交易系统，代码错误直接造成经济损失

## 用户原话

- "为什么更改会覆盖了昨天我们已经改的非常完美的东西？我同一件事做好几次才能做好，改了一遍不行你又给我恢复到错误的原状"
- "你把刚才那点你产生的错误，你修改代码的时候覆盖了以前的代码，把我的代码都给改乱了这一点，给我记下来，以后千万不能再发生了"

## 检查清单（每次改代码前必须过）

1. ✅ 我是否用的 Edit 而不是 Write？
2. ✅ 我是否 Read 了要修改的段落上下文？
3. ✅ 改完后是否立即 git commit？
4. ✅ 我的修改是否只改了目标区域，没有删除其他已有代码？
5. ✅ commit之后立即用 /upload-github 技能推送到GitHub（每次修改都要推，不攒）
