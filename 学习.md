## 参数解析

用户输入的参数 `$ARGUMENTS`，格式：
- 无参数 / `进度`：显示学习进度总览
- `下一课`：从进度中找到下一个"待学"课时，开始讲解
- `第N课` / `课时N`：开始或复习第N课（1-28）
- `第N阶段`：查看第N阶段的详细内容和进度
- `复习`：复习所有已完成课时的要点
- 其他关键词：匹配最相关的课时主题（如 `gamma`、`波动率`、`Kelly`）

## 执行步骤

全程自动化，不要询问确认：

### 1. 读取课程知识库

```bash
cat ~/.claude/projects/-Users-zhangxiaoyu/memory/课程知识库.md
```

获取当前进度和所有课时状态。

### 2. 根据参数执行

#### 模式A：查看进度（无参数 / `进度`）

输出：
- 总进度条（已完成/总课时）
- 每个阶段的完成情况
- 下一个待学课时的主题预告
- 最近完成的课时和学习笔记

#### 模式B：开始课时（`下一课` / `第N课`）

教学流程：
1. **概念讲解**（5-10分钟阅读量）
   - 用通俗语言解释核心概念
   - 数学公式只在必要时给出，优先给直觉
   - 用中国商品期权的真实例子说明

2. **结合你的系统**
   - 读取相关的 trade2026 代码文件
   - 读取 price_sum_knowledge.json 中的相关信念
   - 指出当前系统的对应部分和改进空间

3. **实战演练**
   - 用你的真实数据（parquet/database.db）做计算演示
   - 写代码片段展示概念
   - 如果是编程作业，直接在系统中实现

4. **要点总结**
   - 3-5个关键收获
   - 与已有信念的关联
   - 对交易决策的实际影响

5. **更新进度**
   - 在课程知识库中把该课时状态从"待学"改为"已完成"
   - 在学习笔记区记录关键收获
   - 更新进度条

#### 模式C：复习（`复习`）

汇总所有已完成课时的学习笔记，按阶段分组展示。

#### 模式D：关键词匹配

根据用户输入的关键词，找到最相关的课时：
- `gamma` → 课时1-2
- `波动率` / `vol` / `iv` → 课时5-10
- `iron condor` / `铁鹰` → 课时11
- `kelly` / `仓位` → 课时17
- `bs` / `black scholes` / `定价` → 课时19
- `机器学习` / `ml` → 课时26-27

然后按模式B执行。

### 3. 教学原则

- **从实践到理论**: 你已经有实盘经验，先展示现象再解释原理
- **用你自己的数据**: 所有例子优先用 ag/p/TA 等你熟悉品种的真实数据
- **对接知识库**: 每个概念都关联到 B001-B020 信念体系
- **代码优先**: 能写代码演示的就不只说理论
- **中文为主**: 专业术语给中英文对照，公式用LaTeX
- **控制节奏**: 一次只讲一个课时，讲完等用户消化后再继续
