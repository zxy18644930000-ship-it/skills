# 记录洞察到知识库

将用户的洞察、感悟、经验教训记录到多元思维栅格知识库中。

## 触发方式

用户输入 `/洞察` 后跟一段话，例如：
- "/洞察 今天发现止损设太紧反而亏更多，因为波动是均值回归的"
- "/洞察 查理芒格说过，反过来想，总是反过来想"
- "/洞察 睡眠不足时做的交易决策胜率明显下降"
- "/洞察 读《穷查理宝典》感悟：能力圈不是限制，是保护"

## 执行步骤

1. **读取知识库**: 读取 `~/Scripts/price_sum_knowledge.json`，找到当前最大的信念ID编号（如B024→下一个是B025）

2. **分类**: 根据内容自动判断类别，常见类别包括但不限于：
   - 交易策略 / 风险管理 / 市场结构（期权/交易相关）
   - 健康养生 / 睡眠 / 运动（身体健康）
   - 心理模型 / 认知偏误 / 决策框架（思维方式）
   - 人生哲学 / 价值观 / 人际关系（生活智慧）
   - 读书笔记 / 名人智慧（学习）
   - 物理/数学/经济学/历史（跨学科知识）
   - 如果不属于以上任何类别，创建新类别

3. **构建信念对象**:
```json
{
  "id": "B0XX",
  "category": "自动判断的类别",
  "statement": "用户的洞察内容，整理为清晰的一句话陈述",
  "confidence": 0.50,
  "evidence_for": 1,
  "evidence_against": 0,
  "first_observed": "今天日期",
  "last_updated": "今天日期",
  "data_points": ["用户原始输入 + 上下文补充"],
  "note": "如果有相关的已有信念，在这里注明关联，促进跨领域思考"
}
```

4. **初始置信度规则**:
   - 用户亲身经历/实验验证的 → 0.70
   - 来自可靠来源（书籍/名人/科学研究）→ 0.60
   - 用户的直觉/假设/初步想法 → 0.50
   - 如果用户明确说了置信度，用用户的

5. **跨领域关联**: 检查已有信念，看新洞察是否与某个已有信念存在跨学科关联。例如：
   - 交易中的"止损"↔ 人生中的"及时止损沉没成本"
   - 期权的"theta衰减"↔ 健康中的"年龄增长体能下降是确定的"
   - VRP策略的"均值回归"↔ 心理学的"情绪也会均值回归"
   如果发现关联，在note中标注，并在输出中提醒用户这个跨学科连接

6. **写入知识库**:
   - 将新信念添加到beliefs数组末尾
   - 更新 meta.update_count + 1
   - 添加 update_log 条目
   - 验证JSON合法性

7. **同步**: 复制到 `~/skills/`，commit + push

8. **输出确认**:
```
已记录 B0XX [{类别}]
"{信念陈述}"
置信度: X.XX | 关联: {相关信念ID，如有}
```

## 注意事项

- 如果用户的洞察与已有信念重复或矛盾，不要创建新的，而是更新已有信念（调整置信度、添加evidence）
- 一次输入可以包含多个洞察，分别创建多个信念
- 保持statement简洁有力，一两句话说清楚核心观点
- 知识库是用户的外部大脑，宁可多记不要漏记
