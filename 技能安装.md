从 ClawHub.ai (OpenClaw技能商店) 搜索、预览、安装技能到 Claude Code。

## 参数解析

用户输入的参数 `$ARGUMENTS`，格式：
- **搜索关键词**如 `git` `calendar` `deploy`：搜索相关技能
- **GitHub技能URL**：直接抓取并安装
- **`列表`**：列出已安装的自定义技能
- 无参数：进入交互模式，引导用户选择

## 执行步骤

全程自动化，不要询问确认：

### 1. 搜索技能

根据用户提供的关键词，通过以下方式搜索：

**方式A — Web搜索**：
用 WebSearch 搜索 `clawhub.ai OR openclawskillshub.org "{关键词}" skill`，从搜索结果中找到技能页面链接。

**方式B — GitHub搜索**：
用 WebFetch 抓取 `https://github.com/openclaw/clawhub` 或搜索 `github.com openclaw skill {关键词}`。

**方式C — 用户提供URL**：
直接用 WebFetch 抓取指定URL的 SKILL.md 内容。

### 2. 展示技能列表

搜索到技能后，展示给用户：
```
找到以下技能：
1. [技能名] — 功能描述
2. [技能名] — 功能描述
...
```

如果用户说"列表"，则列出 `~/.claude/commands/` 下所有已安装的 .md 文件。

### 3. 获取技能内容

用 WebFetch 抓取选定技能的 SKILL.md 原始内容。优先尝试 GitHub raw 链接。

### 4. 转换格式

将 SKILL.md 转换为 Claude Code 命令格式：
- **提取**：SKILL.md 的正文部分（跳过 YAML frontmatter 中 OpenClaw 特有的 `metadata.openclaw` 字段）
- **保留**：name、description 作为文件开头的描述行
- **适配**：
  - 将 OpenClaw 特有的工具调用语法替换为 Claude Code 等效操作（Bash/Read/Write/WebFetch等）
  - 如果技能依赖特定 API Key（`requires.env`），在文件开头注明需要配置的环境变量
  - 如果技能依赖特定二进制（`requires.bins`），用 Bash 检查是否已安装，未安装则提示
- **命名**：用技能的 name 字段或 slug 作为文件名，中文技能用中文名

### 5. 安装

将转换后的内容写入 `~/.claude/commands/{技能名}.md`。

安装完成后输出：
```
✅ 技能已安装: {技能名}
📁 路径: ~/.claude/commands/{技能名}.md
🔧 使用方式: /{技能名}
```

如果技能需要环境变量，额外提示：
```
⚠️ 此技能需要配置环境变量:
  - {VAR_NAME}: {用途说明}
```

### 6. 卸载（如果用户要求）

删除 `~/.claude/commands/{技能名}.md` 即可。

## 注意事项

- 安装前检查是否已存在同名技能，已存在则提示覆盖
- 转换时保持技能的核心逻辑不变，只做格式适配
- 如果技能内容太复杂无法自动转换，展示原始内容让用户决定如何处理
- 所有输出用中文
