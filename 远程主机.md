远程主机 - Windows WSL2 服务器 (32核/128G)，用于大规模回测和重型计算。

## 连接
```bash
ssh win    # Tailscale SSH, IP: 100.80.19.31, 用户: jesussos
```
连不上时提醒用户在 WSL2 里跑: `sudo tailscale up --ssh`

## 硬件
- **CPU**: 32核 | **内存**: 128G (WSL2当前分配61G，可调大) | **GPU**: RTX 5080
- **磁盘**: 954G 可用 | **系统**: Ubuntu 24.04 on WSL2
- **Python**: 3.12.3 (pandas 3.0.1, pyarrow 23.0.1, numpy 2.4.3)

## 使用流程

### 1. 检查连接和资源
```bash
ssh win "echo ok && free -h | head -2 && nproc"
```

### 2. 本地编写脚本，路径适配后同步执行
```bash
scp /tmp/script.py win:/home/jesussos/backtest/
ssh win "cd /home/jesussos/backtest && python3 script.py [参数]"
```

### 3. 长时间任务用 tmux
```bash
ssh win "tmux new -d -s bt 'python3 /home/jesussos/backtest/script.py'"
ssh win "tmux attach -t bt"   # 查看进度
```

### 4. 拉取结果到本地可视化
```bash
scp win:/home/jesussos/backtest/result.json /tmp/
```

## 回测分级原则
- **本地 Mac (16G)**: 只跑小规模验证（单品种/少量数据确认逻辑可行）
- **远程 Win (128G)**: 大规模回测（全市场/多参数网格/多品种并行，最多30进程）

## 注意事项
- 远程没有 CTP 实时数据，只用于历史数据回测
- WSL2 重启后需手动: `sudo tailscale up --ssh`
- 仍然遵守 CLAUDE.md 的回测约束（tick分布、止盈止损公式等）
- Windows 文件通过 `/mnt/c/` `/mnt/d/` 访问
- 百度网盘新下载的文件在 `/mnt/d/BaiduNetdiskDownload/`
