## 评估步骤

### Step 1: 加载交易日历计划

读取 `~/Scripts/trading_plan_data.json`：
```python
import json, pandas as pd
from datetime import date

with open('/Users/zhangxiaoyu/Scripts/trading_plan_data.json') as f:
    plan = json.load(f)

today = date.today().isoformat()  # 或最近的交易日
day_plan = plan.get('days', {}).get(today, {})
picks = day_plan.get('picks', [])
```

检查该品种是否在今日计划中。如果在，提取其排名、手数、保证金占比等。

### Step 2: 加载回测结果

读取该品种的回测JSON：
```python
import os
# 品种回测结果
result_path = f'/tmp/{exchange}_{product}_result.json'
if os.path.exists(result_path):
    with open(result_path) as f:
        bt = json.load(f)
```

提取关键指标：
- `n_trades`: 总交易笔数
- `baseline`: 基准策略指标 (wr, sharpe, avg_pnl, max_dd)
- `best_fixed`: 最优固定止盈止损参数
- `best_adaptive`: 最优自适应参数

### Step 3: 加载DTE分析

读取DTE维度分析：
```python
with open('/tmp/dte_analysis_results.json') as f:
    dte_all = json.load(f)
dte_data = dte_all.get(f'{exchange}.{product}', {})
dte_stats = dte_data.get('dte_stats', {})
```

### Step 4: 计算当前DTE

根据品种所属交易所，用知识库中的到期日规则计算当前可交易合约的DTE：
```python
with open('/Users/zhangxiaoyu/Scripts/price_sum_knowledge.json') as f:
    kb = json.load(f)
exp_cal = kb.get('expiration_calendar', {})
```

到期日规则（按交易所）：
- **CZCE**: 交割月前一月15日前(含)倒数第3个交易日
- **DCE**: 交割月前一月第12个交易日
- **SHFE**: 交割月前一月倒数第5个交易日
- **INE**: 交割月前一月倒数第13个交易日
- **GFEX**: 交割月前一月第5个交易日
- **CFFEX**: 交割月第3个周五

计算最近2-3个可交易合约月份的DTE，找出DTE在3-60范围内的合约。

### Step 5: 查询CTP实时数据

从本地数据库获取该品种最新期权数据：

提取：
- 当日成交量和持仓量
- OTM行权价的具体成交量（瓶颈侧）
- 最新权利金报价
- bid/ask价差（如果有）

### Step 6: 综合评分

按以下维度打分（每项0-100分）：

| 维度 | 权重 | 评分依据 |
|------|------|----------|
| **回测质量** | 30% | Sharpe ratio, 胜率, 样本量 |
| **DTE适合度** | 25% | 当前DTE是否落在最优或次优bucket |
| **流动性** | 20% | OTM瓶颈侧成交量, 能容纳的手数 |
| **风险控制** | 15% | 最大回撤, MAE, 止损触发率 |
| **市场环境** | 10% | 近期波幅趋势, 是否临近重大事件 |

**评分标准：**

回测质量:
- Sharpe > 0.15 → 100分
- Sharpe 0.10-0.15 → 80分
- Sharpe 0.05-0.10 → 60分
- Sharpe 0-0.05 → 40分
- Sharpe < 0 → 0分
- 胜率 > 70% → +20分(bonus)

DTE适合度:
- 当前DTE在最优bucket → 100分
- 当前DTE在次优bucket(Sharpe > 0) → 70分
- 当前DTE在可接受bucket(WR > 50%) → 40分
- 当前DTE不在任何profitable bucket → 0分

流动性:
- 瓶颈日成交额 > 500万 → 100分
- 200-500万 → 80分
- 100-200万 → 60分
- 50-100万 → 40分
- < 50万 → 20分

**总分解读：**
- **80-100**: 强烈推荐 ✅ — 可以放心交易，建议满仓位
- **60-79**: 推荐 👍 — 可以交易，建议70%仓位
- **40-59**: 谨慎 ⚠️ — 勉强可交易，建议半仓或更少
- **20-39**: 不推荐 ❌ — 今天不适合，等更好的DTE窗口
- **0-19**: 禁止 🚫 — 该品种回测数据差，不建议交易

## 重要注意事项

1. **数据不全时**: 如果某个数据源缺失，用已有数据评估，但在报告中标注"数据不完整"
2. **50万账户**: 单品种最大保证金不超过10万(20%)，全账户最大75%
3. **DTE优先**: 如果当前DTE完全不在任何profitable bucket，直接判定不适合
4. **流动性硬门槛**: OTM瓶颈侧日成交额 < 30万 → 直接不适合（进出场困难）
5. **回测样本量**: n_trades < 10笔的品种，数据不可靠，降低信心
6. **夜盘时间**: 策略是21:00卖出，次日14:50平仓。评估时注意该品种是否有夜盘
